⑶ P 与 NP 问题

> ① 术语

>> ○ 确定性算法：每一步计算只考虑一种可能性的算法

>> ○ 非确定性算法：在计算的每一步都考虑***两种或更多***可能性的算法

>>> ○ 非确定性算法是一个理论概念，无法在实际计算机上实现

>>> ○ 它可以在每一步同时考虑多个可能的路径，并假设它“幸运地”在其中选择了正确的路径

>>> ○ 你可以想象一个冒险家在每一步都使用克隆技术来寻找最佳路径（[ref](https://gazelle-and-cs.tistory.com/64)）

>>> ○ 因此，即使冒险者没有克隆能力，如果在每一点都有人给出走哪条路的提示，并且该提示可以被验证，那么同样可以搜索最佳路径 

>>> ○ 因此，非确定性算法可以说等价于在多项式时间内验证某些提示的确定性算法

>> ○ **P问题**：=可以通过确定性算法在多项式时间内解决的问题

>>> ○ 将“无法解决问题”解释为“无法通过确定性算法在多项式时间内解决”

>> ○ **NP问题**： = 可以通过非确定性算法在多项式时间内解决的问题 = 可以通过确定性算法在多项式时间内验证的问题

>> ○ **NP-hard**：NP 中的每个问题都是多项式时间多一可简化为另一个问题的问题 

>>> ○ 可以说是比NP问题更难的问题

>> ○ **NP-complete**（NP 完全）：问题 A 属于 NP 的问题集合，同时 NP 中的每个问题都是多项式时间多一可约简到它的。

> ② 定义

>> ○ 由于确定性算法 ⊂ 非确定性算法（例如，每一步有一个正确选择和一个随机选择），因此 P 问题 ⊂ NP 问题

>> ○ P vs NP 问题必须证明 NP 问题是否与 P 问题相同、有更多元素，甚至是否无法证明

>> ○ NP 问题 = (P 问题) + (NP 完全问题) + (非 NP 完全问题，如果 P≠NP)

> ③ 示例

>> ○ 两个城市之间的最短路径问题：P问题。由 Dijkstra、[Floyd 算法](https://jb243.github.io/pages/61) 等解决。

>> ○ 最大公约数计算问题：P问题。通过欧几里德算法求解

>> ○ 【排序问题】(https://jb243.github.io/pages/130)：P问题。通过冒泡排序、快速排序等解决。

>> ○ 素因数分解：尚未证明是NP完全、非NP完全、还是不可证明>> ○ 旅行商问题（TSP，旅行商问题）：NP完全问题。给定多个城市以及每对城市之间的距离，找到访问每个城市一次并返回出发城市的最短路线

>> ○ 片段定向问题（FOP）：NP完全。在生物信息学中，正确确定DNA片段方向的问题

>> ○ 超级马里奥兄弟：NP-hard
